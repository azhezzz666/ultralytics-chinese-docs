## 常见问题

### 什么是 Ultralytics YOLO，它如何使我的[机器学习](https://www.ultralytics.com/glossary/machine-learning-ml)项目受益？

Ultralytics YOLO（You Only Look Once）是一种最先进的实时[目标检测](https://www.ultralytics.com/glossary/object-detection)模型。其最新版本 YOLO11 增强了速度、[准确率](https://www.ultralytics.com/glossary/accuracy)和多功能性，使其非常适合各种应用，从实时视频分析到高级机器学习研究。YOLO 在图像和视频中检测目标的效率使其成为希望将强大[计算机视觉](https://www.ultralytics.com/glossary/computer-vision-cv)功能集成到项目中的企业和研究人员的首选解决方案。

有关 YOLO11 的更多详细信息，请访问 [YOLO11 文档](../models/yolo11.md)。

### 如何为 Ultralytics YOLO 仓库做贡献？

为 Ultralytics YOLO 仓库做贡献非常简单。首先查看[贡献指南](contributing.md)以了解提交拉取请求、报告错误等的协议。您还需要签署[贡献者许可协议 (CLA)](CLA.md)以确保您的贡献得到合法认可。有关有效的错误报告，请参阅[最小可复现示例 (MRE) 指南](minimum-reproducible-example.md)。

### 为什么我应该使用 Ultralytics HUB 进行机器学习项目？

Ultralytics HUB 为管理您的机器学习项目提供了无缝的无代码解决方案。它使您能够轻松生成、训练和部署 YOLO11 等 AI 模型。独特功能包括云训练、实时跟踪和直观的数据集管理。Ultralytics HUB 简化了从数据处理到[模型部署](https://www.ultralytics.com/glossary/model-deployment)的整个工作流程，使其成为初学者和高级用户不可或缺的工具。

要开始使用，请访问 [Ultralytics HUB 快速入门](../hub/quickstart.md)。

### Ultralytics 中的持续集成 (CI) 是什么，它如何确保高质量代码？

Ultralytics 中的持续集成 (CI) 涉及自动化流程，确保代码库的完整性和质量。我们的 CI 设置包括 Docker 部署、断链检查、[CodeQL 分析](https://github.com/github/codeql)和 PyPI 发布。这些流程通过自动运行新代码提交的测试和检查来帮助维护稳定和安全的仓库。

在[持续集成 (CI) 指南](CI.md)中了解更多信息。

### Ultralytics 如何处理[数据隐私](https://www.ultralytics.com/glossary/data-privacy)？

Ultralytics 非常重视数据隐私。我们的[隐私政策](privacy.md)概述了我们如何收集和使用匿名数据来改进 YOLO 包，同时优先考虑用户隐私和控制。我们遵守严格的数据保护法规，确保您的信息始终安全。

有关更多信息，请查看我们的[隐私政策](privacy.md)。
