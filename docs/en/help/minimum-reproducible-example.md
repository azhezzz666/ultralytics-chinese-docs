# 为错误报告创建最小可复现示例

在向 [Ultralytics](https://www.ultralytics.com/) [YOLO](https://github.com/ultralytics) 仓库提交错误报告时，提供[最小可复现示例 (MRE)](https://stackoverflow.com/help/minimal-reproducible-example) 至关重要。MRE 是一小段独立的代码，用于演示您遇到的问题。提供 MRE 有助于维护者和贡献者理解问题并更高效地修复它。本指南解释了在向 Ultralytics YOLO 仓库提交错误报告时如何创建 MRE。

## 1. 隔离问题

创建 MRE 的第一步是隔离问题。删除与问题不直接相关的任何不必要的代码或依赖项。专注于导致问题的代码的特定部分，并消除任何无关部分。

## 2. 使用公共模型和数据集

创建 MRE 时，使用公开可用的模型和数据集来复现问题。例如，使用 `yolo11n.pt` 模型和 `coco8.yaml` 数据集。这确保维护者和贡献者可以轻松运行您的示例并调查问题，而无需访问专有数据或自定义模型。

## 3. 包含所有必要的依赖项

确保 MRE 中包含所有必要的依赖项。如果您的代码依赖于外部库，请指定所需的包及其版本。理想情况下，如果您安装了 `ultralytics`，请使用 `yolo checks` 在错误报告中列出依赖项，或使用 `pip list` 列出其他工具。

## 4. 清楚描述问题

提供您遇到的问题的清晰简洁描述。解释预期行为和您遇到的实际行为。如果适用，包括任何相关的错误消息或日志。

## 5. 正确格式化代码

在问题描述中使用代码块正确格式化代码。这使其他人更容易阅读和理解您的代码。在 GitHub 中，您可以通过用三个反引号 (\```) 包裹代码并指定语言来创建代码块：

````bash
```python
# 您的 Python 代码放在这里
```
````

## 6. 测试您的 MRE

在提交 MRE 之前，测试它以确保它准确地复现问题。确保其他人可以在没有任何问题或修改的情况下运行您的示例。

## MRE 示例

以下是假设错误报告的 MRE 示例：

**错误描述：**

在对 0 通道图像运行推理时，我收到与输入张量维度相关的错误。

**MRE：**

```python
import torch

from ultralytics import YOLO

# 加载模型
model = YOLO("yolo11n.pt")

# 加载 0 通道图像
image = torch.rand(1, 0, 640, 640)

# 运行模型
results = model(image)
```

**错误消息：**

```
RuntimeError: Expected input[1, 0, 640, 640] to have 3 channels, but got 0 channels instead
```

**依赖项：**

- `torch==2.3.0`
- `ultralytics==8.2.0`

在此示例中，MRE 用最少的代码演示了问题，使用了公共模型 (`"yolo11n.pt"`)，包含了所有必要的依赖项，并提供了问题的清晰描述以及错误消息。

通过遵循这些指南，您将帮助 Ultralytics YOLO 仓库的维护者和[贡献者](https://github.com/ultralytics/ultralytics/graphs/contributors)更高效地理解和解决您的问题。
