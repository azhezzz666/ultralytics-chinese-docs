| 参数            | 类型             | 默认值                 | 描述                                                                                                                                                                                                                                                                                                    |
| --------------- | ---------------- | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `source`        | `str`            | `'ultralytics/assets'` | 指定推理的数据源。可以是图像路径、视频文件、目录、URL 或实时流的设备 ID。支持多种格式和来源，可灵活应用于[不同类型的输入](https://docs.ultralytics.com/modes/predict/#inference-sources)。                                                                                                               |
| `conf`          | `float`          | `0.25`                 | 设置检测的最小置信度阈值。置信度低于此阈值的检测对象将被忽略。调整此值可以帮助减少误检。                                                                                                                                                                                                                |
| `iou`           | `float`          | `0.7`                  | 非极大值抑制（NMS）的[交并比](https://www.ultralytics.com/glossary/intersection-over-union-iou)（IoU）阈值。较低的值通过消除重叠框来减少检测数量，有助于减少重复检测。                                                                                                                                   |
| `imgsz`         | `int` 或 `tuple` | `640`                  | 定义推理的图像尺寸。可以是单个整数 `640` 用于正方形调整大小，或 (height, width) 元组。适当的尺寸可以提高检测[精度](https://www.ultralytics.com/glossary/accuracy)和处理速度。                                                                                                                            |
| `rect`          | `bool`           | `True`                 | 如果启用，最小化填充图像的短边直到可被步幅整除以提高推理速度。如果禁用，在推理期间将图像填充为正方形。                                                                                                                                                                                                  |
| `half`          | `bool`           | `False`                | 启用半[精度](https://www.ultralytics.com/glossary/precision)（FP16）推理，可以在支持的 GPU 上加速模型推理，同时对精度影响最小。                                                                                                                                                                         |
| `device`        | `str`            | `None`                 | 指定推理设备（例如 `cpu`、`cuda:0` 或 `0`）。允许用户在 CPU、特定 GPU 或其他计算设备之间选择执行模型。                                                                                                                                                                                                   |
| `batch`         | `int`            | `1`                    | 指定推理的批量大小（仅当源是[目录、视频文件或 `.txt` 文件](https://docs.ultralytics.com/modes/predict/#inference-sources)时有效）。较大的批量大小可以提供更高的吞吐量，缩短推理所需的总时间。                                                                                                            |
| `max_det`       | `int`            | `300`                  | 每张图像允许的最大检测数量。限制模型在单次推理中可以检测的对象总数，防止在密集场景中输出过多。                                                                                                                                                                                                          |
| `vid_stride`    | `int`            | `1`                    | 视频输入的帧步幅。允许跳过视频中的帧以加快处理速度，但会牺牲时间分辨率。值为 1 处理每一帧，更高的值跳过帧。                                                                                                                                                                                              |
| `stream_buffer` | `bool`           | `False`                | 确定是否为视频流排队传入帧。如果为 `False`，旧帧会被丢弃以容纳新帧（针对实时应用优化）。如果为 `True`，在缓冲区中排队新帧，确保不跳过帧，但如果推理 FPS 低于流 FPS 会导致延迟。                                                                                                                          |
| `visualize`     | `bool`           | `False`                | 在推理期间激活模型特征的可视化，提供模型"看到"什么的洞察。用于调试和模型解释。                                                                                                                                                                                                                          |
| `augment`       | `bool`           | `False`                | 启用预测的测试时增强（TTA），可能以推理速度为代价提高检测鲁棒性。                                                                                                                                                                                                                                        |
| `agnostic_nms`  | `bool`           | `False`                | 启用类别无关的非极大值抑制（NMS），合并不同类别的重叠框。在多类别检测场景中类别重叠常见时很有用。                                                                                                                                                                                                        |
| `classes`       | `list[int]`      | `None`                 | 将预测过滤为一组类别 ID。只返回属于指定类别的检测。在多类别检测任务中专注于相关对象时很有用。                                                                                                                                                                                                            |
| `retina_masks`  | `bool`           | `False`                | 返回高分辨率分割掩码。如果启用，返回的掩码（`masks.data`）将匹配原始图像尺寸。如果禁用，它们具有推理期间使用的图像尺寸。                                                                                                                                                                                 |
| `embed`         | `list[int]`      | `None`                 | 指定从哪些层提取特征向量或[嵌入](https://www.ultralytics.com/glossary/embeddings)。用于聚类或相似性搜索等下游任务。                                                                                                                                                                                     |
| `project`       | `str`            | `None`                 | 如果启用 `save`，保存预测输出的项目目录名称。                                                                                                                                                                                                                                                           |
| `name`          | `str`            | `None`                 | 预测运行的名称。用于在项目文件夹内创建子目录，如果启用 `save`，预测输出将存储在此处。                                                                                                                                                                                                                   |
| `stream`        | `bool`           | `False`                | 通过返回 Results 对象的生成器而不是一次性将所有帧加载到内存中，实现长视频或大量图像的内存高效处理。                                                                                                                                                                                                     |
| `verbose`       | `bool`           | `True`                 | 控制是否在终端显示详细的推理日志，提供预测过程的实时反馈。                                                                                                                                                                                                                                              |
| `compile`       | `bool` 或 `str`  | `False`                | 启用 PyTorch 2.x `torch.compile` 图编译，使用 `backend='inductor'`。接受 `True` → `"default"`、`False` → 禁用，或字符串模式如 `"default"`、`"reduce-overhead"`、`"max-autotune-no-cudagraphs"`。如果不支持则回退到 eager 模式并发出警告。                                                               |
