| 参数           | 类型            | 默认值  | 描述                                                                                                                                                                                                                                                                             |
| -------------- | --------------- | ------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `data`         | `str`           | `None`  | 指定数据集配置文件的路径（例如 `coco8.yaml`）。此文件应包含[验证数据](https://www.ultralytics.com/glossary/validation-data)的路径。                                                                                                                                               |
| `imgsz`        | `int`           | `640`   | 定义输入图像的尺寸。所有图像在处理前都会调整到此尺寸。较大的尺寸可能提高小物体的精度，但会增加计算时间。                                                                                                                                                                          |
| `batch`        | `int`           | `16`    | 设置每批的图像数量。较高的值更有效地利用 GPU 内存，但需要更多 VRAM。根据可用硬件资源进行调整。                                                                                                                                                                                    |
| `save_json`    | `bool`          | `False` | 如果为 `True`，将结果保存到 JSON 文件以供进一步分析、与其他工具集成或提交到 COCO 等评估服务器。                                                                                                                                                                                   |
| `conf`         | `float`         | `0.001` | 设置检测的最小置信度阈值。较低的值增加召回率但可能引入更多误检。在[验证](https://docs.ultralytics.com/modes/val/)期间用于计算精确率-召回率曲线。                                                                                                                                  |
| `iou`          | `float`         | `0.7`   | 设置[非极大值抑制](https://www.ultralytics.com/glossary/non-maximum-suppression-nms)的[交并比](https://www.ultralytics.com/glossary/intersection-over-union-iou)阈值。控制重复检测的消除。                                                                                       |
| `max_det`      | `int`           | `300`   | 限制每张图像的最大检测数量。在密集场景中有用，可防止过多检测并管理计算资源。                                                                                                                                                                                                      |
| `half`         | `bool`          | `True`  | 启用半[精度](https://www.ultralytics.com/glossary/precision)（FP16）计算，减少内存使用并可能提高速度，同时对[精度](https://www.ultralytics.com/glossary/accuracy)影响最小。                                                                                                       |
| `device`       | `str`           | `None`  | 指定验证设备（`cpu`、`cuda:0` 等）。当为 `None` 时，自动选择最佳可用设备。可以用逗号分隔指定多个 CUDA 设备。                                                                                                                                                                       |
| `dnn`          | `bool`          | `False` | 如果为 `True`，使用 [OpenCV](https://www.ultralytics.com/glossary/opencv) DNN 模块进行 ONNX 模型推理，提供 [PyTorch](https://www.ultralytics.com/glossary/pytorch) 推理方法的替代方案。                                                                                          |
| `plots`        | `bool`          | `False` | 当设置为 `True` 时，生成并保存预测与真实值的对比图、混淆矩阵和 PR 曲线，用于可视化评估模型性能。                                                                                                                                                                                  |
| `classes`      | `list[int]`     | `None`  | 指定要训练的类别 ID 列表。用于在评估期间过滤并只关注某些类别。                                                                                                                                                                                                                    |
| `rect`         | `bool`          | `True`  | 如果为 `True`，使用矩形推理进行批处理，减少填充并可能通过以原始宽高比处理图像来提高速度和效率。                                                                                                                                                                                   |
| `split`        | `str`           | `'val'` | 确定用于验证的数据集分割（`val`、`test` 或 `train`）。允许灵活选择用于性能评估的数据段。                                                                                                                                                                                          |
| `project`      | `str`           | `None`  | 保存验证输出的项目目录名称。帮助组织不同实验或模型的结果。                                                                                                                                                                                                                        |
| `name`         | `str`           | `None`  | 验证运行的名称。用于在项目文件夹内创建子目录，存储验证日志和输出。                                                                                                                                                                                                                |
| `verbose`      | `bool`          | `False` | 如果为 `True`，在验证过程中显示详细信息，包括每类指标、批次进度和额外调试信息。                                                                                                                                                                                                   |
| `save_txt`     | `bool`          | `False` | 如果为 `True`，将检测结果保存到文本文件，每张图像一个文件，用于进一步分析、自定义后处理或与其他系统集成。                                                                                                                                                                         |
| `save_conf`    | `bool`          | `False` | 如果为 `True`，当启用 `save_txt` 时在保存的文本文件中包含置信度值，为分析和过滤提供更详细的输出。                                                                                                                                                                                 |
| `workers`      | `int`           | `8`     | 数据加载的工作线程数。较高的值可以加速数据预处理，但可能增加 CPU 使用。设置为 0 使用主线程，在某些环境中可能更稳定。                                                                                                                                                              |
| `augment`      | `bool`          | `False` | 在验证期间启用测试时增强（TTA），可能以推理速度为代价通过对输入的变换版本运行推理来提高检测精度。                                                                                                                                                                                 |
| `agnostic_nms` | `bool`          | `False` | 启用类别无关的[非极大值抑制](https://www.ultralytics.com/glossary/non-maximum-suppression-nms)，无论预测类别如何都合并重叠框。用于以实例为中心的应用。                                                                                                                           |
| `single_cls`   | `bool`          | `False` | 在验证期间将所有类别视为单一类别。用于评估二分类检测任务的模型性能或当类别区分不重要时。                                                                                                                                                                                          |
| `visualize`    | `bool`          | `False` | 可视化每张图像的真实值、真阳性、假阳性和假阴性。用于调试和模型解释。                                                                                                                                                                                                              |
| `compile`      | `bool` 或 `str` | `False` | 启用 PyTorch 2.x `torch.compile` 图编译，使用 `backend='inductor'`。接受 `True` → `"default"`、`False` → 禁用，或字符串模式如 `"default"`、`"reduce-overhead"`、`"max-autotune-no-cudagraphs"`。如果不支持则回退到 eager 模式并发出警告。                                        |
